# QRSPI Research

Announce: "I'm using QRSPI Research to answer the approved research questions."

## Required input

- Approved `questions.md`

Do not give research subagents `goals.md`; keep research as objective as practical.

## Steps

1. Read `questions.md`.
2. For each question or related group, research with the appropriate tools.
3. Save optional per-question notes as `research/qNN.md`.
4. Synthesize `research/summary.md` without relying on goals or intended solution.
5. Include:
   - facts found
   - file paths / code references
   - external docs links when relevant
   - open questions
   - constraints discovered
6. Present `research/summary.md` for approval.
7. On approval, mark it `status: approved`.
8. Next phase:
   - full route: Design
   - quick route: Plan

## Quality bar

Research should answer the questions, not propose the implementation plan. Keep design opinions for the Design or Plan phase.
